## Context

ADR 0013 defers four mechanisms to this design round: the exported-surface
annotation, the munging requirements handed to the M2 emitter as a public
contract, go.mod versioning for emitted libs, and the c-shared header shape.
The emitter's compilation model (design/04 §1: one ns → one Go package;
munging conventions in §6; go.mod management in §2) is the substrate. The
M2 emitter change (`m2-emitter-v0`, if open in openspec/changes/) owns the
emitter implementation — this change hands it requirements and layers the
library product surface on top; it never edits that change.

## Goals / Non-Goals

**Goals:**
- One codebase → Clojure lib / Go lib / C lib / exe with zero restructuring.
- Emitted Go a Go developer would accept in review: real signatures where
  hinted, doc comments, stable names.
- Choose the munging scheme once, publicly, before M2 freezes it.

**Non-Goals:**
- Registry/resolver, WASM, C++ bindings, stability for unannotated vars.

## Decisions

### D1 — Exported-surface annotation (settled)
Primary: **`^:export` metadata on the var** (`(defn ^:export parse [s] …)`).
Secondary: project-file key `:cljgo/exports {namespace [sym …]}` that ADDS
exports without touching source (for generated or vendored .clj). Rules:
export set = union of both; exporting requires the var to exist at compile
time (positioned error otherwise); un-exported vars emit unexported Go
identifiers (lowercase first rune post-munge). `-main` is implicitly
exported for exe builds only. Alternatives rejected: exports-map-only (far
from the code, rots), export-everything-public (freezes internals into the
public contract, violating the stability guarantee ADR 0013 demands).

### D2 — Munging requirements handed to the M2 emitter (settled as a contract; implementation lives in m2-emitter-v0)
The emitter's munging becomes a **public, versioned contract from M2**
(ADR 0013 consequences). Requirements handed over:
1. Deterministic and total: same symbol → same Go identifier, forever;
   documented as a table in user docs.
2. Baseline scheme per design/04 §6: JVM-compatible char munges (`-`→`_`,
   `?`→`_QMARK_`, `!`→`_BANG_`, `*`→`_STAR_`), ns→path `.`→`/` `-`→`_`
   (design/04 §1), leading `_` gets `X` prefix.
3. Export casing layer ON TOP for `^:export` vars: exported Go identifier is
   the munged name with first rune uppercased and kebab segments
   camel-cased (`parse-config` → `ParseConfig`); collisions after casing
   (`parse-config` vs `parseConfig`) are a compile error naming both vars.
4. Round-trip documented: every emitted exported identifier carries a
   `// cljgo: my-ns/parse-config` provenance comment.
5. Any future munge change = major version of the emitted-API contract.
Verification per config rule: casing/identifier legality checked against Go
source (go/token identifier rules), not priors.

### D3 — go.mod versioning for emitted libs (settled)
Module path comes from the project file (`:go-module "github.com/me/lib"`;
required for `--lib`, error if absent). `cljgo build --lib` writes a
self-contained module dir (default `target/lib/`): emitted .go files, go.mod
(module path + go 1.26 + pinned `github.com/muthuishere/cljgo` runtime
requirement at the compiler's own version), LICENSE passthrough, and a
`// Code generated by cljgo <version>; DO NOT EDIT.` header per file.
Publishing = the user commits/tags that emitted source in a repo (go get
needs VCS-hosted source; there is no artifact registry to push to). Semver
is the user's tag; cljgo's contribution is determinism — same input, same
bytes — so regenerating for a patch produces a reviewable diff. Runtime
version skew rule: emitted code pins the exact runtime version it was
generated against; consumers get it transitively.

### D4 — c-shared header shape (settled)
`--c-shared`/`--c-archive` use Go's own buildmodes (Gloat precedent, ADR
0013). Export eligibility: `^:export` vars whose hinted signature is
C-expressible — integers/floats/bool, `*C.char` strings (copied, callee
never retains), `[]byte` as ptr+len pairs. Everything else: excluded from
the C surface with a build warning naming the var and the offending type
(NOT an error — the Go surface can still export it). Mechanism: pkg/emit
generates `//export <Name>` cgo wrapper fns; Go's buildmode emits
`<lib>.h` automatically. We additionally ship a hand-curated
`cljgo_runtime.h` declaring `CljgoInit()` (runs all Load()s; must be called
before any export; idempotent) and `CljgoShutdown()`. Boxed-`any` C export
via opaque handles is explicitly deferred (Non-goal for v1) — the 80% case
is hinted leaf fns.

## Risks / Trade-offs

- [Casing collisions surface late] → compile-time collision error (D2.3) at emit, not at go build.
- [Pinned runtime version forces lockstep upgrades on consumers] → deliberate: version skew between emitted code and runtime is untested space; document.
- [c-shared needs cgo + platform toolchains] → already mandated first-class (design/00 §1.5); document zig cc escape hatch.
- [Emitted-source publishing feels alien to Clojure users] → docs lead with the "it's just a Go module" story; CI recipe provided.

## Open Questions

- Whether `--lib` should also emit a `doc.go` package overview from the ns docstring — decide during implementation (cheap either way).
- Exact `:go-module` project-file location depends on the project-file format settled by the M2 emitter change — tracked there.
