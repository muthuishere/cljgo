# ADR 0011 — C FFI: purego dlopen as the primary door, cgo as the ecosystem door
Date: 2026-07-12 · Status: accepted (design/05; spike S7; owner mandate priority 5)

## Context
The C ecosystem must be reachable (owner: "any ffi can be imported and used
directly"). Two mechanisms exist: cgo (compile-time, needs a C toolchain,
robust, cross-compile pain) and purego (runtime dlopen/dlsym, no C toolchain,
REPL-friendly). S7 proved purego on darwin/arm64 end-to-end against system
sqlite — including Go callbacks into C — with CGO_ENABLED=0, cross-compiling
cleanly; it also proved variadic C functions silently corrupt on the Apple
arm64 ABI.

## Decision
1. **`ffi/deflib` on purego is the primary FFI**: declarative binding form —
   `(ffi/deflib sqlite "libsqlite3" (version [] :string) (open [path :string
   db* :ptr*] :int) …)` — usable LIVE at the REPL (dlopen needs no rebuild).
   Type mapping table owned by pkg/host; C int return codes and out-params
   shape into the ADR 0005 [v err]/`!` convention.
2. **Variadic C functions are rejected at deflib expansion time** with a
   positioned error naming the S7 ABI finding. No silent wrong answers.
3. **cgo is the second door and is first-class**: cgo-based Go modules
   (sqlite drivers, GUI/audio) flow through ordinary ADR 0010 interop —
   nothing special to do. `cljgo build` supports CGO_ENABLED=1, passes CC/CXX
   and build tags through, documents zig-cc as the cross-compile escape
   hatch. Pure-Go remains the default for painless cross-compilation.
4. Callbacks via purego NewCallback are supported within its platform limits
   (documented per-platform; no-string-args/slot limits noted from S7).

## Consequences
C libraries are reachable interactively (REPL dlopen) AND industrially (cgo
modules), with one error convention across Go and C. deflib implementation
targets post-M3 via OpenSpec; the emitted-code path embeds the same purego
bindings so dual-mode consistency holds. Platform ABI caveats live in the
deflib reference doc, enforced by expansion-time checks, not runtime faith.
