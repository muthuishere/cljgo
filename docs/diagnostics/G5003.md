# G5003 — value is not seqable

**Message pattern:** `can't convert <type> to ISeq`

## Cause

A sequence operation (`seq`, `first`, `rest`, `next`, `count`, `map`,
`reduce`, destructuring, …) received a value that cannot be turned into a
sequence. Seqable values are collections, strings, nil, and native seqs;
a number, keyword, or boolean is not.

On the JVM this surfaces as an `IllegalArgumentException` — *Don't know
how to create ISeq from: java.lang.Long*.

## Example

```clojure
(first 5)        ; can't convert int64 to ISeq
(map inc 42)     ; 42 is not seqable
```

## Fix

Wrap the value in a collection (`[5]`, `(list 5)`) or fix the argument
order — a scalar where a collection was expected usually means two
arguments were swapped, or a producing call returned a number instead of
a collection.
