# G5004 — index out of bounds

**Message patterns:** `index <n> out of bounds` · `index out of bounds`

## Cause

An indexed access (`nth`, vector/subvec indexing, `aget`) used an index
outside the collection's `[0, count)` range. Unlike `get`, which returns
`nil` (or a supplied default) for a missing index, `nth` throws when the
index is out of range and no default is given.

On the JVM this surfaces as an `IndexOutOfBoundsException`.

## Example

```clojure
(nth [1 2] 9)    ; index 9 out of bounds
(nth [] 0)       ; empty vector has no index 0
```

## Fix

Supply a default to make the access total — `(nth coll i not-found)` or
`(get coll i default)` — or bounds-check with `(< i (count coll))` first.
Remember indices are zero-based.
