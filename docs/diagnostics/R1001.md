# R1001 — unterminated form

**Message pattern:** `EOF while reading <list|vector|map|set>, expected "<delim>" to close it, starting at line <L> column <C>`
(also covers `EOF while reading` for strings, characters, dispatch, metadata, and wrapped forms)

## Cause

The input ended in the middle of a form: an opening delimiter `(`, `[`,
`{`, `#{`, or an opening `"` was never closed. The reader tracks where
the open form started and reports both positions — the point of EOF and
the start of the unterminated form (surfaced as a `related` note).

## Example

```clojure
(defn add [a b
  (+ a b))
```

The vector opened at `[a b` is never closed with `]` — the `(` of
`(+ ...)` is read as a vector element, and input ends with the vector
(and both lists) still open.

## Fix

Go to the start position named in the diagnostic and add the matching
closing delimiter. Balanced-paren editor support (paredit, parinfer)
prevents this class of error entirely. In a REPL, this error means the
form is incomplete — keep typing rather than re-entering it.
