# A2002 — recur outside tail position

**Message patterns:** `can only recur from tail position` · `cannot recur across <try|catch|finally>`

## Cause

`recur` must be the last thing its loop or fn evaluates — the tail
position. A `recur` whose result would still be used by an enclosing
expression (an argument to `+`, a non-final form in a body, inside
`try`) cannot be compiled as a jump. It is also reported when there is
no enclosing `loop`/`fn` at all, or when the recur target is on the
other side of a `try`/`catch`/`finally` boundary.

## Example

```clojure
(defn fact [n]
  (if (zero? n)
    1
    (* n (recur (dec n)))))   ; recur is an argument to *, not a tail call
```

## Fix

Restructure so `recur` is the final expression — usually by carrying
the partial result in a loop binding:

```clojure
(defn fact [n]
  (loop [n n, acc 1]
    (if (zero? n) acc (recur (dec n) (* acc n)))))
```

If no loop or fn encloses the `recur`, wrap the iteration in `loop`.
Across `try`, hoist the `try` inside the loop body or restructure so
the recur does not cross it.
