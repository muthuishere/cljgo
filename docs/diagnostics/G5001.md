# G5001 — value is not a number

**Message pattern:** `cannot convert <type> to Ops`

## Cause

A numeric operation (`+`, `-`, `*`, `/`, `inc`, `dec`, `<`, comparisons,
…) received an argument that is not a number. Clojure's numeric tower
dispatches on the argument's type; a string, keyword, nil, or collection
has no numeric operations, so the coercion to the internal `Ops` fails.

On the JVM this surfaces as a `ClassCastException` — *class
java.lang.String cannot be cast to class java.lang.Number*.

## Example

```clojure
(inc "x")     ; cannot convert string to Ops
(+ 1 :a)      ; cannot convert keyword to Ops
```

## Fix

Pass a number. Parse strings first (`(parse-long s)` / `(Integer/parseInt
s)`), and guard values that may be `nil` before doing arithmetic.
