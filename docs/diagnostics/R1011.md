# R1011 — conditional read not allowed

**Message patterns:** `Conditional read not allowed`

## Cause

`read-string` met a reader conditional (`#?(...)` / `#?@(...)`) without
being given permission to process it. Mirroring JVM Clojure,
`clojure.core/read-string` only reads conditionals when its opts map
carries `{:read-cond :allow}` (select a branch) or
`{:read-cond :preserve}` (return the conditional as data).

Note: cljgo FILE and REPL reading always processes reader conditionals,
in any source extension — a documented divergence from the JVM's
`.cljc`-only gate (ADR 0068 addendum). This error is raised only by
`read-string`.

## Example

```clojure
(read-string "#?(:cljgo 1)")
;; Conditional read not allowed
```

## Fix

Pass an explicit `:read-cond` mode:

```clojure
(read-string {:read-cond :allow} "#?(:cljgo 1)")     ;=> 1
(read-string {:read-cond :preserve} "#?(:clj 1)")    ;=> #?(:clj 1)
(read-string {:read-cond :allow :features #{:clj}}
             "#?(:clj 1 :cljs 2)")                    ;=> 1
```
