# A2007 — invalid binding form

**Message patterns:** `bad binding form, expected symbol, got: <form>` · `can't let qualified name: <sym>` · `can't bind name containing a period: <sym>`

## Cause

The name side of a binding must be a plain, unqualified symbol. This
error fires when it is a literal (keyword, number, string), when the
symbol is namespace-qualified (`ns/x`), or when it contains a period.
At the core `let*`/`fn*` level even destructuring vectors/maps are
rejected — destructuring is macro-expanded away before analysis, so
seeing it here usually means a macro emitted the form unexpanded.

## Example

```clojure
(let [:x 1] ...)        ; bad binding form, expected symbol, got: :x
(let [foo/x 1] ...)     ; can't let qualified name: foo/x
```

## Fix

Bind plain local names: `(let [x 1] ...)`. Drop the namespace
qualifier — locals are always unqualified — and avoid periods in
local names. If a macro produced this, make sure it emits symbols in
binding position (or expands destructuring itself).
