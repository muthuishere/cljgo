# A2008 — conflicting fn overloads

**Message patterns:** `can't have 2 overloads with same arity` · `can't have more than 1 variadic overload` · `can't have fixed arity function with more params than variadic function`

## Cause

A multi-arity `fn`/`defn` declares overloads that cannot be dispatched
unambiguously: two bodies with the same parameter count, more than one
variadic (`& rest`) body, or a fixed-arity body that takes more
required parameters than the variadic body — a call with that many
arguments could match either.

## Example

```clojure
(fn f
  ([x] x)
  ([y] (inc y)))          ; can't have 2 overloads with same arity

(fn g
  ([a & more] a)
  ([a b c] b))            ; fixed arity exceeds the variadic's params
```

## Fix

Give each overload a distinct arity, keep at most one variadic body,
and make the variadic body's required params at least as many as the
largest fixed arity. If two same-arity bodies differ by argument
*type*, dispatch inside one body (or use a multimethod/protocol) —
fn overloads dispatch on count only.
