# G5007 — no value supplied for key

**Message patterns:** `no value supplied for key: <k>` ·
`invalid map. must have even number of inputs`

## Cause

A map constructor or `assoc`-style call received an odd number of key/value
arguments, so the final key has no value to pair with. Maps are built from
alternating key value key value …; a dangling trailing key is a mistake.

On the JVM this surfaces as an `IllegalArgumentException` reading
`No value supplied for key: <k>`.

## Example

```clojure
(hash-map :a 1 :b)      ; :b has no value
(array-map :a 1 :b)     ; :b has no value
(sorted-map :a 1 :b)    ; :b has no value
{:a 1 :b}               ; reader-level: odd number of forms in a map literal
```

Note the map *literal* `{:a 1 :b}` is caught earlier, by the reader
(see `R1003`); `G5007` is the runtime constructor path (`hash-map`,
`array-map`, `sorted-map`, `struct-map`).

## Fix

Supply the missing value, or check that every key is paired:
`(hash-map :a 1 :b 2)`. When building a map from a computed sequence, make
sure it has an even length — `(apply hash-map (interleave ks vs))` with
`(= (count ks) (count vs))`.
