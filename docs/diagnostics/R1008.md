# R1008 — invalid character literal

**Message patterns:** `Unsupported character: \<token>` · `Invalid character constant: \u<hex>` · `Invalid unicode character: \<token>` · `Invalid character length: <n>, should be: <m>` · `Invalid digit: <c>`

## Cause

A character literal `\...` is not a single character or a recognized
name. Valid forms: `\a` (any single char), named characters
(`\newline \space \tab \backspace \formfeed \return`), unicode
`\uXXXX` (four hex digits, not in the surrogate range D800–DFFF), and
octal `\oNNN` (up to three octal digits, ≤ 0377).

## Example

```clojure
\newlin      ; Unsupported character: \newlin
\u12         ; Invalid character length: 2, should be: 4
\ud800       ; Invalid character constant: \ud800
```

## Fix

Check the spelling of named characters, pad unicode escapes to four
hex digits, and keep octal escapes in range. For a literal string of
several characters, use a `"string"` instead of a character literal.
