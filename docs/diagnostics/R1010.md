# R1010 — reader conditional splicing at top level

**Message patterns:** `Reader conditional splicing not allowed at the top level.`

## Cause

A splicing reader conditional `#?@(...)` appeared at the top level of a
file (or REPL input). Splicing merges the branch's elements into a
SURROUNDING collection — with no surrounding form there is nothing to
splice into, so the reader rejects it. This matches JVM Clojure exactly.

## Example

```clojure
#?@(:cljgo [(defn f [] 1) (defn g [] 2)])
;; Reader conditional splicing not allowed at the top level.
```

## Fix

Use one conditional per top-level form:

```clojure
#?(:cljgo (defn f [] 1))
#?(:cljgo (defn g [] 2))
```

or wrap the multiple forms in a `do`:

```clojure
#?(:cljgo (do (defn f [] 1)
              (defn g [] 2)))
```
