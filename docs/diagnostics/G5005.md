# G5005 — value is not a collection

**Message pattern:** `conj: cannot conj onto <value>`

## Cause

`conj` (and `into`, which builds on it) requires its first argument to be
a persistent collection — a list, vector, map, or set — or `nil` (which
seeds a list). A number, string, or keyword is not a collection, so there
is nothing to conj onto.

On the JVM this surfaces as a `ClassCastException` — *class
java.lang.Long cannot be cast to class clojure.lang.IPersistentCollection*.

## Example

```clojure
(conj 5 1)         ; conj: cannot conj onto 5
(into 0 [1 2 3])   ; 0 is not a collection
```

## Fix

Start from an actual collection: `(conj [] 1)`, `(conj #{} 1)`, or
`(into [] xs)`. A common cause is threading a scalar accumulator where a
collection was intended — check the seed value of a `reduce`/`into`.
