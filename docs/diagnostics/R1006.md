# R1006 — invalid number literal

**Message pattern:** `Invalid number: <token>`

## Cause

A token starting with a digit (or `+`/`-` digit) failed to parse as
any numeric literal: integer, float, ratio, radix (`2r1010`, `16rFF`),
hex (`0x1F`), octal (`017`), bigint (`42N`), or bigdecimal (`1.5M`).
A trailing letter or misplaced character is the usual culprit —
anything that starts with a digit must be a number, never a symbol.

## Example

```clojure
12abc     ; Invalid number: 12abc
3.4.5     ; Invalid number: 3.4.5
8r9       ; digit out of range for radix 8
```

## Fix

Fix the literal's syntax, or — if you meant a symbol — rename it so it
does not start with a digit. Check radix prefixes (`NrDIGITS` with
digits valid for base N) and that only one `.` and one exponent appear
in floats.
