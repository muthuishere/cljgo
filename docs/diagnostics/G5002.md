# G5002 — value is not a function

**Message pattern:** `cannot apply non-function <type>`

## Cause

The head of a call form evaluated to something that is not callable — a
number, string, keyword used positionally, nil, or a collection. In
Clojure `(f a b)` requires `f` to implement `IFn`; a plain value does not.

On the JVM this surfaces as a `ClassCastException` — *class
java.lang.Long cannot be cast to class clojure.lang.IFn*.

## Example

```clojure
(1 2 3)            ; cannot apply non-function int64
(let [x 10] (x))   ; x is a number, not a function
```

## Fix

Check the call head. A common cause is an extra pair of parentheses
(`((f x))` when you meant `(f x)`), or shadowing a function name with a
local bound to a value. Keywords and maps *are* callable — `(:k m)` and
`(m :k)` — but numbers and strings are not.
