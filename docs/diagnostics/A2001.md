# A2001 — unable to resolve symbol

**Message patterns:** `unable to resolve symbol: <sym> in this context` · `unable to resolve var: <sym> in this context`

## Cause

A symbol names nothing in scope: it is not a local binding, not a var
interned in the current namespace, and not reachable through a
referred or aliased namespace. Typical causes: a typo, a missing
`(:require ...)` / `refer`, using a binding outside the form that
introduced it, or calling a function before it is defined (Clojure
resolves top-level forms in order).

## Example

```clojure
(defn area [r] (* pie r r))   ; unable to resolve symbol: pie
```

## Fix

Check the spelling against the intended var; add the missing
`:require` (with the right alias) to the `ns` form; move the
definition above its first use, or forward-declare it with
`(declare name)`. If the symbol was meant to be a local, confirm you
are inside the `let`/`fn` that binds it.
