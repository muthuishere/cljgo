# R1004 — duplicate key in map or set literal

**Message pattern:** `Duplicate key: <key>`

## Cause

A map literal `{...}` contains the same key twice, or a set literal
`#{...}` contains the same element twice. Literal collections reject
duplicates at read time (matching JVM Clojure) because a duplicate is
almost always an editing mistake — the later entry would silently win.

## Example

```clojure
{:a 1 :a 2}   ; Duplicate key: :a
#{1 2 1}      ; Duplicate key: 1
```

## Fix

Remove or rename the duplicated key/element. If you genuinely need
"last write wins" semantics, build the map at runtime instead:
`(hash-map :a 1 :a 2)` or `(assoc {:a 1} :a 2)`.
