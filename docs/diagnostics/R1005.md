# R1005 — invalid token

**Message pattern:** `Invalid token: <token>` (also `Invalid token: ##<form>`, `Invalid token (auto-resolved keyword requires a resolver): <token>`)

## Cause

A token could not be read as a symbol, keyword, or other valid form.
Typical causes: a keyword with a trailing colon (`:foo:`), a bare `::`
without an alias resolver in scope, malformed namespace syntax
(`foo//bar`), or a `##` symbolic-value dispatch that is not followed
by a symbol.

## Example

```clojure
:foo:        ; Invalid token: :foo:
##"not-sym"  ; Invalid token: ##"not-sym"
```

## Fix

Correct the token's spelling to a valid symbol/keyword shape. For
auto-resolved keywords (`::k`, `::alias/k`), make sure they are read
in a namespace context where the alias resolves. For `##`, only the
known symbolic values (`##Inf`, `##-Inf`, `##NaN`) are valid.
