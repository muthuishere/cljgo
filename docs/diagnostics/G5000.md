# G5000 — uncategorized compiler error

**Message pattern:** any compiler error message not yet mapped to a
specific code.

## Cause

The compiler reported an error that has no dedicated code in the
diagnostics registry yet. The structured-diagnostics adapter
(`pkg/diag.FromCompilerError`) classifies known message shapes into
R/A/E/I-band codes; anything it does not recognize is surfaced under
G5000 so it is still machine-readable — with the original message and
best-effort position — rather than dropped.

## Example

Any error whose message matches none of the registered patterns, e.g.
a newly added analyzer check that has not been assigned a code yet.

## Fix

Read the diagnostic's `message` field — it is the compiler's full
human-readable explanation and states the actual problem. If you hit
G5000 for a common, recurring error, that is a signal the error
deserves its own registry code: file an issue (or add the code —
append-only — with an explain page under `docs/diagnostics/`).
