# A2004 — wrong number of forms in special form

**Message patterns:** `too many arguments to <if|def>` · `too few arguments to if` · `wrong number of args (<n>) passed to <quote|var>`

## Cause

A special form received the wrong number of forms: `if` takes a test,
a then-branch, and an optional else-branch (2–3 forms); `def` takes a
name, an optional docstring, and an optional init (1–3 forms);
`quote` and `var` take exactly one form. A common cause is an extra
expression that was meant to be grouped — special forms have fixed
shapes and do not take "bodies".

## Example

```clojure
(if (pos? x) :yes :no :maybe)   ; too many arguments to if
(if (pos? x))                   ; too few arguments to if
```

## Fix

Match the form's shape. To run several expressions in a branch, group
them with `do`: `(if t (do a b) c)` — or use `when`/`when-not` when
there is no else-branch. For `def`, check that a stray form has not
slipped between the name and the init.
