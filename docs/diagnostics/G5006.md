# G5006 — divide by zero

**Message patterns:** `Divide by zero` · `/ by zero`

## Cause

An integer or exact division had a zero divisor. Unlike floating-point
division, which yields `##Inf`/`##-Inf`/`##NaN`, integer and rational
arithmetic has no representation for the result, so it throws.

On the JVM this surfaces as an `ArithmeticException`. The two message texts
are both divide-by-zero and are oracle-frozen against JVM Clojure 1.12.5
(`conformance/tests/divide-by-zero-messages.clj`):

- `Divide by zero` — `/`, `mod`, and bigint `quot`/`rem`.
- `/ by zero` — fixnum `quot`/`rem` (a JVM primitive-inlining artifact; the
  same underlying `ArithmeticException`).

## Example

```clojure
(/ 1 0)        ; Divide by zero
(/ 0 0)        ; Divide by zero
(quot 1 0)     ; / by zero
(rem 1 0)      ; / by zero
(mod 10N 0N)   ; Divide by zero
```

## Fix

Guard the divisor before dividing — `(when-not (zero? d) (/ n d))` — or use
floating-point division if an infinity/NaN result is acceptable:
`(/ 1.0 0.0)` yields `##Inf` rather than throwing.
