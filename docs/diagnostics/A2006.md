# A2006 — malformed binding vector

**Message patterns:** `<let*|loop*|binding> requires a binding vector` · `<form> requires a vector for its bindings` · `<form> requires an even number of forms in binding vector`

## Cause

Binding forms (`let`, `loop`, `binding`, and friends) take a vector of
name/value pairs as their first argument. This error fires when the
bindings are missing, are not a vector (e.g. a list), or contain an
odd number of forms — a name with no value expression.

## Example

```clojure
(let [x 1 y] (+ x y))    ; odd number of forms: y has no value
(let (x 1) x)            ; bindings must be a vector, not a list
```

## Fix

Give every name exactly one value expression, in pairs:
`(let [x 1, y 2] ...)`. Commas are whitespace and can make pairs
easier to see. If a value expression spans lines, check a delimiter
did not close early and swallow the following name.
