# A2005 — def name is not a symbol

**Message pattern:** `first argument to def must be a symbol`

## Cause

The first form after `def` must be a plain symbol naming the var to
create. Anything else — a keyword, a string, a list, a vector, or an
expression that would compute a name — is rejected: vars are created
at compile time, so their names cannot be dynamic.

## Example

```clojure
(def :answer 42)        ; first argument to def must be a symbol
(def (str "x" 1) 42)    ; names cannot be computed
```

## Fix

Use a symbol: `(def answer 42)`. To create vars with computed names,
use `intern` at runtime; for a keyword-keyed value, use a map. If this
came from a macro, check the macro emits a symbol (not a quoted list
or keyword) in the name position.
