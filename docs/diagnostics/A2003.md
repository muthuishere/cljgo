# A2003 — recur argument count mismatch

**Message pattern:** `mismatched argument count to recur, expected: <n> args, got: <m>`

## Cause

`recur` rebinds its target's bindings positionally, so it must pass
exactly as many arguments as the enclosing `loop` has bindings (or the
enclosing fn method has parameters). Passing fewer or more means some
binding would be left unset or have no destination.

## Example

```clojure
(loop [i 0, acc []]
  (if (< i 3)
    (recur (inc i))       ; expected: 2 args, got: 1
    acc))
```

## Fix

Pass one argument per binding, in binding order — here
`(recur (inc i) (conj acc i))`. If a binding never changes across
iterations, it still must be passed (pass it through unchanged), or
lift it out of the `loop` so it is closed over instead of bound.
Note that a variadic fn's rest param counts as one argument to
`recur` (pass a seq).
