# R1007 — invalid escape sequence in string

**Message patterns:** `Unsupported escape character: \<c>` · `Invalid unicode escape: \u<c>` · `Octal escape sequence must be in range [0, 377]` · `Invalid octal escape sequence length: <n>`

## Cause

A backslash escape inside a `"string"` is not one of the supported
forms: `\t \r \n \\ \" \b \f`, a unicode escape `\uXXXX` (exactly four
hex digits), or an octal escape `\0`–`\377`.

## Example

```clojure
"bad \q escape"    ; Unsupported escape character: \q
"\u12G4"           ; Invalid unicode escape: \uG
"\400"             ; Octal escape sequence must be in range [0, 377]
```

## Fix

Use a supported escape, double the backslash for a literal one
(`"C:\\path"`), or complete the unicode escape to four hex digits.
For regular expressions prefer `#"pattern"` literals, which do not
process string escapes.
